# iBreeze 用户手册

> 最后更新：2026-07-21
> 项目：iBreeze AI 公司运行平台

## 功能概述

iBreeze 是一个本地 AI 组织运行平台，采用三层架构：

1. **桌面应用（Desktop）**：基于 Tauri 构建，提供 9 个终端用户页面，涵盖公司管理、员工管理、会话、任务看板与概览等核心功能。
2. **管理后端（Admin Backend）**：基于 FastAPI（端口 50080），提供管理 REST API，支撑所有管理数据的增删改查。
3. **管理前端（Admin Frontend）**：基于 React + AntDesign（端口 8000），提供 11 个管理页面，涵盖能力、技能、提示词、模板、知识库、治理审批、Provider 与 Backend、审计与人工干预等。

所有数据存储在本地（SQLite + LanceDB），无需联网。

## 左侧导航（桌面应用）

侧边栏共 7 个菜单项，点击切换：

1. 公司管理（/companies）
2. 员工管理（/employees）
3. 会话（/session）— 默认页面
4. 任务看板（/tasks）
5. 任务高级（/tasks/advanced）
6. 概览（/dashboard）— 轻量概览
7. 设置（/settings）

顶部折叠按钮可收起/展开侧边栏（收起后仅显示图标）。每个页面右上角均有「刷新」按钮，点击重新加载当前页面数据。

## 公司管理

公司管理是系统的核心入口，每个公司代表一个独立的 AI 组织单元。

- 查看所有公司列表（名称、状态、创建时间）
- 新建公司
- 查看公司详情

**操作：** 点击「公司管理」→ 右上角「新建公司」→ 点击列表中的公司行查看详情。

---

## 员工管理

员工是公司内的 AI 角色，每个员工有明确职责与类型。

- 查看员工列表（姓名、角色、**基座模型**、状态）
- 新建员工（可绑定「基座模板」指定底层 Agent / 模型 / Provider）
- 查看员工详情

**操作：** 点击「员工管理」→「新建员工」→ 填写姓名/角色/所属公司/部门 → 选择「基座模板」（决定该员工使用的 Provider 与模型，如 opencode + big-pickle）→ 确认。列表的「基座模型」列直接展示 `模型 · Provider`。

---

## 会话

会话是职员（员工）基于「安全上下文」的对话线程。系统按九维安全上下文（三个策略域）计算密钥，同一上下文自动复用线程；任一维度变化则新建线程，且跨上下文不共享原始记忆。

- 查看会话列表（按员工筛选）
- 查看会话详情
- 新建会话（指定员工与安全上下文参数）
- 跨员工交接（handoff）
- 在活动回合（active turn）内发送消息（并发占用会提示忙）

**操作：** 点击「会话」→ 选择员工 → 新建会话或点击已有会话 → 在详情中发送消息。

---

## 任务看板

以三列视图展示任务进度：待处理、进行中、已完成。

- 看板分组查看任务
- 查看任务优先级与详情
- 任务可触发工作流（并行执行、审批挂起、成本降级）

**操作：** 点击「任务看板」→ 对应列查看卡片 → 点击卡片查看任务详情。

---

## 概览

平台轻量概览页面，汇总展示关键指标与最近任务。

- 概览统计卡片
- 最近任务列表

**操作：** 点击「概览」查看概览；点击任务可跳转任务详情。

## 设置

桌面应用的轻量设置页面，提供基础配置项。

**操作：** 点击「设置」→ 修改配置并保存。

## 管理前端（Admin Frontend）

管理前端通过浏览器访问，地址为 `http://localhost:8000`，需登录后使用。提供以下 11 个管理页面：

### 登录（Login）

管理员登录入口，输入凭证后进入管理后台。

### 能力管理（Capabilities）

管理可复用的能力资产，支持 CRUD 操作与状态流转（草稿 → 已发布 → 已归档）。

### 技能管理（Skills）

管理技能定义，支持 CRUD 操作与状态流转。

### 提示词管理（Prompts）

管理提示词模板，支持 CRUD 操作与状态流转。

### 模板管理（Templates）

管理任务/流程模板，支持 CRUD 操作与状态流转。

### 能力引擎（Capability Engine）

查看能力引擎的健康状态，监控运行情况。

### 知识库（Knowledge）

管理公司文档与资料，支持 CRUD 操作。文档支持治理确认（confirm）或驳回（reject）操作。

### 知识治理规则（Knowledge Governance）

配置知识库的治理规则，支持 CRUD 操作。

### 治理审批类型（Governance）

管理审批类型定义，支持 CRUD 操作。

### Provider 与 Backend

管理 AI 提供方（Provider）与其后端执行环境（Backend），支持 CRUD 操作与状态流转。

- **Provider**：注册 API Key 形式或调用 Agent 形式的提供方。
- **Backend**：注册后端执行环境，查看健康状态与租用（lease）状态。

### 审计日志（Audit Log）

查看治理相关审计记录，包括预算、审批、审批类型、预算修订等条目，只读。

### 人工干预（Interventions）

查看需要人工处理的事项（如 Backend 恢复 `backend_recovery` 等），按公司隔离，只读。

## 通用功能说明

### 时间显示

系统中所有时间均显示为**北京时间（UTC+8）**。带 `Z` 或显式时区的时间先换算到东八区后展示；无时区的后端时间按北京时间墙钟展示，避免浏览器本地时区偏移。

### 数值显示

所有数值使用统一格式化：默认最多保留 2 位小数，且不补零。涉及金额时以「元」展示（后端以 int64 micros 存储）。

### 刷新数据

每个页面右上角有「刷新」按钮，点击重新加载当前页面数据。

### 侧边栏

点击顶部折叠按钮可收起/展开侧边栏；收起后仅显示图标，鼠标悬停可识别。
