# iBreeze 用户手册

> 最后更新：2026-07-19
> 项目：iBreeze AI 公司桌面应用

## 功能概述

iBreeze 是一个本地 AI 组织运行平台，用于管理 AI 驱动的公司、员工、任务、知识库与治理审批。所有数据存储在本地（SQLite + LanceDB），无需联网。平台通过统一的 JSON-RPC 接口驱动后端 sidecar，前端所有页面通过左侧导航切换。

---

## 左侧导航（全部页面）

侧边栏共 15 个页面，点击切换：

1. Dashboard
2. 公司管理
3. 员工管理
4. 任务看板
5. 知识库
6. 能力管理（能力 / 提示词 / 技能 / 模板）
7. 员工模板
8. 会话
9. Provider 与 Backend
10. 授权
11. 人工干预
12. 审计
13. 设置

顶部折叠按钮可收起/展开侧边栏（收起后仅显示图标）。每个页面右上角均有「刷新」按钮，点击重新加载当前页面数据。

---

## 公司管理

公司管理是系统的核心入口，每个公司代表一个独立的 AI 组织单元。

- 查看所有公司列表（名称、状态、创建时间）
- 新建公司
- 查看公司详情

**操作：** 点击「公司管理」→ 右上角「新建公司」→ 点击列表中的公司行查看详情。

---

## 员工管理

员工是公司内的 AI 角色，每个员工有明确职责与类型。

- 查看员工列表（姓名、角色、**基座模型**、状态）
- 新建员工（可绑定「基座模板」指定底层 Agent / 模型 / Provider）
- 查看员工详情

**操作：** 点击「员工管理」→「新建员工」→ 填写姓名/角色/所属公司/部门 → 选择「基座模板」（决定该员工使用的 Provider 与模型，如 opencode + big-pickle）→ 确认。列表的「基座模型」列直接展示 `模型 · Provider`。

---

## 任务看板

以三列视图展示任务进度：待处理、进行中、已完成。

- 看板分组查看任务
- 查看任务优先级与详情
- 任务可触发工作流（并行执行、审批挂起、成本降级）

**操作：** 点击「任务看板」→ 对应列查看卡片 → 点击卡片查看任务详情。

---

## 知识库

用于存储与管理公司相关的文档与资料，并支持 RAG 检索。

- 查看文档列表（标题、来源分类、状态）
- 添加新文档
- 查看文档详情
- 检索与引用（ACL 按公司/部门/任务/员工隔离）

**操作：** 点击「知识库」→「添加文档」→ 点击文档行查看详情。

---

## 能力管理

管理可复用的能力资产，包含四个子页签：

- **能力**：已注册的能力列表
- **提示词**：提示词模板
- **技能**：技能定义
- **模板**：任务/流程模板

**操作：** 点击「能力管理」→ 切换上方页签查看对应资产。

---

## 员工模板

员工模板（EmployeeTemplate）是员工的「基座配置」，定义该员工底层使用的 Provider（如 opencode）、模型（如 big-pickle）与绑定的能力。员工创建时选择模板即可继承这套基座配置。

- 查看模板列表（角色、模型、**Provider**、能力版本、状态）
- 新建模板（填写能力 ID / 版本 / 默认角色 / 模型，并选择 Provider）
- 编辑草稿模板、激活（启用）、归档

**操作：** 点击「员工模板」→「新建模板」→ 填写能力 ID、默认角色 → 选择 Provider（从已注册的 Provider 列表中拉取）→ 填写模型名 → 创建（状态为草稿）→ 点击播放按钮激活。激活后的模板方可在「员工管理」中绑定给员工。

---

## 会话

会话是职员（员工）基于「安全上下文」的对话线程。系统按九维安全上下文（三个策略域）计算密钥，同一上下文自动复用线程；任一维度变化则新建线程，且跨上下文不共享原始记忆。

- 查看会话列表（按员工筛选）
- 查看会话详情
- 新建会话（指定员工与安全上下文参数）
- 跨员工交接（handoff）
- 在活动回合（active turn）内发送消息（并发占用会提示忙）

**操作：** 点击「会话」→ 选择员工 → 新建会话或点击已有会话 → 在详情中发送消息。

---

## Provider 与 Backend

管理 AI 提供方（Provider）与其后端执行环境（Backend）。

- **Provider**：注册/查看 Provider（名称、类型、状态）；查看版本化价格目录、运行时能力。点击「新建 Provider」填写名称与类型（如 opencode / agent）即可注册，供员工模板绑定使用。
- **Backend**：新建/查看/更新/归档 Backend；手动健康探测；查看租用（lease）状态。点击「新建 Backend」填写名称与类型（local_process / openai / anthropic）注册。

**操作：** 点击「Provider 与 Backend」→ 点「新建 Provider」或「新建 Backend」注册 → 在列表中管理（探测、启用、排空、归档等）。

---

## 授权

授权用于向员工授予能力（capability），遵循权限引擎 ACL（同部门、跨部门、employee_private、task_private 四类范围，只收窄不放大）。

- 查看授权列表
- 新建授权（选择员工与能力）
- 查看/撤销授权

**操作：** 点击「授权」→「新建授权」→ 填写员工与能力 → 确认；列表中可撤销。

---

## 人工干预

列出需要人工处理的事项（如 Backend 恢复 `backend_recovery` 等），按公司隔离并分页。

- 查看干预事项列表（类型、状态、时间）
- 处理/标记干预事项

**操作：** 点击「人工干预」→ 查看列表 → 对需要处理的项进行操作。

---

## 审计

查询治理相关的审计记录，支持按类型分页：预算（budget）、审批（approval）、审批类型（approval_type）、预算修订（budget_revision）。

- 选择审计类型
- 分页浏览审计条目（时间、主体、对象、动作）

**操作：** 点击「审计」→ 选择类型 → 浏览分页结果。

---

## Dashboard

平台概览页面，汇总展示关键指标与最近任务。

- 概览统计卡片
- 最近任务列表

**操作：** 点击「Dashboard」查看概览；点击任务可跳转任务详情。

---

## 设置

管理版本化的平台策略与云端授权：

- 四类策略（权限 / 成本 / 安全 / 审计），均带版本号（CAS 更新）
- 云端使用授权（consent）查看与更新
- 功能开关（features）

**操作：** 点击「设置」→ 切换策略类别 → 编辑并保存（需保持版本一致）。

## 通用功能说明

### 时间显示

系统中所有时间均显示为**北京时间（UTC+8）**。带 `Z` 或显式时区的时间先换算到东八区后展示；无时区的后端时间按北京时间墙钟展示，避免浏览器本地时区偏移。

### 数值显示

所有数值使用统一格式化：默认最多保留 2 位小数，且不补零。涉及金额时以「元」展示（后端以 int64 micros 存储）。

### 刷新数据

每个页面右上角有「刷新」按钮，点击重新加载当前页面数据。

### 侧边栏

点击顶部折叠按钮可收起/展开侧边栏；收起后仅显示图标，鼠标悬停可识别。
