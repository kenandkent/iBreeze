# iBreeze 部署文档

## 本地开发部署

### 后台服务

```bash
# 启动 PostgreSQL
docker run -d --name ibreeze-postgres \
  -e POSTGRES_DB=ibreeze \
  -e POSTGRES_USER=ibreeze \
  -e POSTGRES_PASSWORD=dev_password \
  -p 5432:5432 \
  postgres:16

# 启动 MinIO
docker run -d --name ibreeze-minio \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -p 9000:9000 \
  -p 9001:9001 \
  minio/minio server /data --console-address ":9001"

# 运行数据库迁移
cd apps/backend-api
uv run alembic upgrade head

# 启动后端 API
uv run uvicorn ibreeze_backend.main:app --reload --host 0.0.0.0 --port 8000

# 启动管理后台 UI
cd ../admin-web
npm run dev
```

### 桌面客户端

```bash
# 开发模式
cd apps/desktop
npm run dev

# Tauri 开发模式（需要 Rust）
cd apps/desktop-core
cargo tauri dev
```

## 生产部署

### 后台服务

使用 Docker Compose 部署：

```bash
cd deploy
docker compose up -d
```

### 桌面客户端

1. 构建 macOS 安装包：
```bash
cd apps/desktop-core
cargo tauri build
```

2. 签名和公证（需要 Apple Developer 证书）

## 环境变量

### 后台 API

| 变量 | 说明 | 默认值 |
|---|---|---|
| DATABASE_URL | PostgreSQL 连接串 | - |
| S3_ENDPOINT | MinIO/S3 端点 | - |
| S3_ACCESS_KEY | S3 访问密钥 | - |
| S3_SECRET_KEY | S3 密钥 | - |
| JWT_SECRET | JWT 签名密钥 | - |
| CATALOG_SIGNING_KEY | 目录签名私钥 | - |

### 桌面客户端

桌面客户端通过 Tauri 配置文件设置，不使用环境变量。
