# iBreeze 部署文档

## 系统架构

| 服务 | 用途 | 端口 |
|---|---|---|
| postgres:16 | 数据库 | 51543 |
| minio/minio | S3 对象存储 | 51900 (API) / 51901 (Console) |
| backend-api | FastAPI 后端 | 51080 |
| admin-web | React 管理后台 (Vite) | 51421 (dev) |

## 一键部署（推荐）

### 后台服务

```bash
# 1. 构建并启动所有服务
docker compose up -d

# 2. 创建 MinIO 存储桶
docker exec ibreeze-minio-1 mc alias set local http://localhost:9000 minioadmin minioadmin
docker exec ibreeze-minio-1 mc mb local/ibreeze

# 3. 运行数据库迁移
cd apps/backend-api
IBREEZE_DATABASE_URL="postgresql+asyncpg://ibreeze:ibreeze_password@localhost:51543/ibreeze" \
  uv run alembic upgrade head

# 4. 验证服务健康
curl http://localhost:51080/health
# 预期: {"status":"ok"}

# 查看实时日志
docker compose logs -f
```

### 管理后台前端

```bash
cd apps/admin-web
npm install
npm run dev
# 访问 http://localhost:51421
```

## 本地分步启动

### 数据库和存储

```bash
# PostgreSQL
docker run -d --name ibreeze-postgres \
  -e POSTGRES_DB=ibreeze \
  -e POSTGRES_USER=ibreeze \
  -e POSTGRES_PASSWORD=ibreeze_password \
  -p 51543:5432 \
  postgres:16

# MinIO
docker run -d --name ibreeze-minio \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  -p 51900:9000 \
  -p 51901:9001 \
  minio/minio server /data --console-address ":51901"

# 创建 MinIO 存储桶
docker exec ibreeze-minio-1 mc alias set local http://localhost:9000 minioadmin minioadmin
docker exec ibreeze-minio-1 mc mb local/ibreeze
```

### 后端 API (开发模式)

```bash
cd apps/backend-api
uv sync
IBREEZE_DATABASE_URL="postgresql+asyncpg://ibreeze:ibreeze_password@localhost:51543/ibreeze" \
  uv run alembic upgrade head
IBREEZE_DATABASE_URL="postgresql+asyncpg://ibreeze:ibreeze_password@localhost:51543/ibreeze" \
  uv run uvicorn ibreeze_backend.main:app --reload --host 0.0.0.0 --port 51080
```

### 管理后台 UI (开发模式)

```bash
cd apps/admin-web
npm install
npm run dev
```

### 桌面客户端

```bash
# 开发模式
cd apps/desktop
npm run dev

# 构建安装包（需要 Rust，自动清理多余构建产物）
bash scripts/build-desktop.sh

# Tauri 开发模式（需要 Rust）
cd apps/desktop-core
cargo tauri dev
```

## 常见运维

### 查看日志

```bash
docker compose logs -f backend-api   # 仅后端
docker compose logs -f postgres      # 仅数据库
docker compose logs -f minio         # 仅对象存储
```

### 停止/重启

```bash
docker compose down        # 停止所有服务
docker compose up -d       # 重新启动
docker compose restart     # 重启
```

### 完全重置（清除数据）

```bash
docker compose down -v     # 停止并删除卷（数据会丢失）
docker compose up -d       # 重新创建
# 之后重新创建 MinIO 存储桶和运行迁移
```

### 从零重新部署

```bash
# 卸载旧版
docker compose down -v
docker system prune -a --volumes -f

# 清理本地缓存
rm -rf apps/backend-api/.venv sidecar/.venv apps/admin-web/node_modules apps/desktop/node_modules
pip cache purge
uv cache clean
npm cache clean --force

# 重新安装
cd apps/backend-api && uv sync
cd ../admin-web && npm install

# 构建 Docker 镜像
cd ../..
docker compose build
docker compose up -d
# 然后创建存储桶 + 迁移数据库
```

## 环境变量

### 后台 API

| 变量 | 说明 | 默认值 |
|---|---|---|
| IBREEZE_DATABASE_URL | PostgreSQL 连接串 | `postgresql+asyncpg://ibreeze:ibreeze@localhost:51543/ibreeze` |
| IBREEZE_API_PORT | 后端 API 监听端口 | `51080` |
| IBREEZE_S3_ENDPOINT_URL | MinIO/S3 端点 | `http://localhost:51900` |
| IBREEZE_S3_ACCESS_KEY_ID | S3 访问密钥 | `minioadmin` |
| IBREEZE_S3_SECRET_ACCESS_KEY | S3 密钥 | `minioadmin` |
| IBREEZE_S3_BUCKET_NAME | S3 存储桶名 | `ibreeze` |
| IBREEZE_JWT_SECRET | JWT 签名密钥 | - |
| IBREEZE_CATALOG_SIGNING_KEY | 目录签名私钥 | - |

### 桌面客户端

桌面客户端通过 Tauri 配置文件设置，不使用环境变量。
