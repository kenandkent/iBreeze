# iBreeze 部署文档

> 最后更新：2026-07-21
> 项目：iBreeze AI 公司桌面应用

---

## 1. 系统要求

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Node.js | 18+ | 前端依赖与 Vite 构建 |
| npm | 9+ | 前端包管理 |
| Python | 3.12+ | Sidecar + Admin Backend |
| uv | 最新 | Python 依赖管理（替代 pip） |
| Rust | 稳定版（含 `cargo`） | Tauri 2 桌面壳编译 |
| 操作系统 | macOS / Windows / Linux | Tauri 目标平台 |

> 所有数据存储于本地（SQLite + LanceDB），默认无需联网。

## 4. 数据库迁移

### 4.1 Sidecar 迁移

后端使用 SQLite，迁移脚本位于 `sidecar/migrations/`（命名格式 `YYYYMMDDHHMMSS_description.sql`）。迁移幂等（`CREATE TABLE IF NOT EXISTS` / 条件 `ADD COLUMN`）。

启动时 `acos.app` 自动执行未完成迁移。手动检查迁移状态：

```bash
# 通过 RPC（应用运行后）
rpcCall('sys.migration.status', {})
```

### 4.2 Admin Backend 迁移

Admin Backend 使用 Alembic 管理迁移，迁移脚本位于 `admin-backend/migrations/`。

```bash
cd admin-backend
uv run alembic upgrade head
```

## 5. 端口分配

| 端口 | 服务 | 用途 |
|------|------|------|
| UDS (`/tmp/acos.sock`) | Sidecar | 桌面端 ↔ Sidecar JSON-RPC 通信 |
| 50080 | Admin Backend | REST API（管理侧） |
| 50081 | Sidecar Internal | 内部服务端口 |
| 8000 | Admin Frontend | 管理界面 dev server |

## 6. 本地开发部署

本节适用于开发者在本地机器上运行全部服务。

### 6.1 仅桌面端（无管理功能）

仅运行 Sidecar + 桌面端，Admin Backend 不参与。

```bash
# 终端 1：Sidecar
cd sidecar
uv run python -m acos.app

# 终端 2：桌面端
cd apps/desktop
npm run dev
```

### 6.2 完整开发环境（含 Admin Backend + Frontend）

运行全部四个服务，Admin Frontend 通过 Vite proxy 转发 `/api` 到 Admin Backend。

```bash
# 终端 1：Sidecar
cd sidecar
uv run python -m acos.app

# 终端 2：Admin Backend
cd admin-backend
uv run uvicorn app.main:app --port 50080

# 终端 3：Admin Frontend
cd apps/admin
npm run dev

# 终端 4：桌面端
cd apps/desktop
npm run dev
```

Admin Frontend 配置（`apps/admin/vite.config.ts`）已内置 dev proxy：

```ts
server: {
  port: 8000,
  proxy: {
    '/api': { target: 'http://127.0.0.1:50080', changeOrigin: true },
  },
},
```

### 6.3 本地部署 Admin Backend 为独立服务

可将 Admin Backend 作为后台进程运行，用于非桌面端场景的 API 调试：

```bash
cd admin-backend
uv run uvicorn app.main:app --port 50080 --host 127.0.0.1
```

### RPC 链路

```
Desktop App → UDS JSON-RPC → Sidecar
                                    ↓ (REST Pull)
                            Admin Backend (port 50080)
                                    ↑
                            Admin Frontend (port 8000)
```

## 7. 线上/生产部署

本节适用于将服务部署到生产服务器。本地开发请参见第 6 节。

### 7.1 环境变量配置

Admin Backend 通过 `IBREEZE_ADMIN_` 前缀读取环境变量（定义于 `admin-backend/app/config.py`）：

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `IBREEZE_ADMIN_JWT_SECRET_KEY` | `ibreeze-admin-secret-change-in-production` | JWT 签名密钥，**生产环境必须更换！** |
| `IBREEZE_ADMIN_ADMIN_DB_PATH` | `/tmp/ibreeze_admin.db` | SQLite 数据库路径 |
| `IBREEZE_ADMIN_HOST` | `127.0.0.1` | 监听地址 |
| `IBREEZE_ADMIN_PORT` | `50080` | 监听端口 |

> ⚠️ 生产环境必须设置 `IBREEZE_ADMIN_JWT_SECRET_KEY` 为随机强密钥，否则认证形同虚设。
> ⚠️ 不要将数据库放在 `/tmp`，重启后数据丢失。

```bash
export IBREEZE_ADMIN_JWT_SECRET_KEY=$(openssl rand -hex 32)
export IBREEZE_ADMIN_ADMIN_DB_PATH=/var/lib/ibreeze/admin.db
export IBREEZE_ADMIN_HOST=0.0.0.0
```

### 7.2 Admin Backend 生产部署

使用 uvicorn 多 worker 模式运行：

```bash
cd admin-backend
uv run uvicorn app.main:app --host 0.0.0.0 --port 50080 --workers 4
```

推荐使用 systemd 或 supervisor 管理进程。systemd 服务示例（`/etc/systemd/system/ibreeze-admin.service`）：

```ini
[Unit]
Description=iBreeze Admin Backend
After=network.target

[Service]
Type=simple
User=ibreeze
WorkingDirectory=/opt/ibreeze/admin-backend
Environment=IBREEZE_ADMIN_JWT_SECRET_KEY=<your-production-secret>
Environment=IBREEZE_ADMIN_ADMIN_DB_PATH=/var/lib/ibreeze/admin.db
Environment=IBREEZE_ADMIN_HOST=0.0.0.0
Environment=IBREEZE_ADMIN_PORT=50080
ExecStart=/opt/ibreeze/admin-backend/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 50080 --workers 4
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable ibreeze-admin
sudo systemctl start ibreeze-admin
```

### 7.3 Admin Frontend 构建与部署

构建静态文件：

```bash
cd apps/admin
npm run build   # 输出到 dist/
```

将 `dist/` 目录部署到 nginx 或任意静态文件服务器。

nginx 配置示例（前端 + API 反向代理）：

```nginx
server {
    listen 443 ssl;
    server_name admin.example.com;

    ssl_certificate     /etc/ssl/ibreeze-admin.crt;
    ssl_certificate_key /etc/ssl/ibreeze-admin.key;

    # Admin Frontend 静态文件
    location / {
        root /var/www/ibreeze-admin/dist;
        try_files $uri $uri/ /index.html;
    }

    # Admin Backend API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:50080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 7.4 Sidecar 部署

Sidecar 以 **per-user** 模式运行，随桌面应用一同打包分发，无需单独部署。

- 桌面端安装后，Sidecar 二进制位于应用内部
- 启动时由 Tauri 自动拉起 Sidecar 进程
- 通信通过 UDS（`/tmp/acos.sock`），无网络暴露

### 7.5 CORS 配置

Admin Backend 当前硬编码允许 `http://localhost:8000`（`admin-backend/app/main.py`）。

生产环境需修改 `allow_origins` 为实际域名：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://admin.example.com"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

或通过环境变量配置（需修改代码支持）。

### 7.6 安全注意事项

- **HTTPS**：生产环境必须使用 TLS，Admin Backend 不直接暴露端口
- **JWT Secret**：使用 `openssl rand -hex 32` 生成，定期轮换
- **数据库**：SQLite 文件权限设为 `0600`，限制访问用户
- **网络隔离**：Admin Backend 监听 `0.0.0.0` 时，仅通过 nginx 反向代理暴露，不直接对外
- **凭证存储**：Provider 凭证存于 OS Keychain，明文不落 SQLite

## 11. 常见问题

### 开发环境

- **迁移报错**：确认 `sidecar/migrations/` 下脚本未被手动改动；修复后重跑 `uv run python -m acos.app` 即可。
- **端口 / socket 占用**：Sidecar 使用 UDS，确保应用数据目录可写。
- **Admin Backend 启动失败**：确认 `admin-backend/app/config.py` 中端口 50080 未被占用。
- **Admin Frontend 401**：确认 Admin Backend 已启动且 CORS 已配置（允许 `localhost:8000`）。

### 生产环境

- **JWT 认证失败**：确认 `IBREEZE_ADMIN_JWT_SECRET_KEY` 已设置且前后端使用相同密钥。
- **数据库路径不存在**：确保 `IBREEZE_ADMIN_ADMIN_DB_PATH` 指向的目录已创建且可写。
- **CORS 被浏览器拦截**：确认 nginx 反向代理正确传递了 Origin 头，且 Admin Backend 的 `allow_origins` 包含生产域名。
- **uvicorn worker 数量**：建议 `workers = 2 * CPU核心数 + 1`，过高会增加内存占用。

### 通用

- **凭证存储**：Provider 凭证存于 OS Keychain，明文不落 SQLite。
