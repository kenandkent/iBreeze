# iBreeze 部署文档

> 最后更新：2026-07-19
> 项目：iBreeze AI 公司桌面应用
> 适用范围：本地开发、桌面应用打包发布

---

## 1. 系统要求

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Node.js | 18+ | 前端依赖与 Vite 构建 |
| npm | 9+ | 前端包管理 |
| Python | 3.12+ | 后端 sidecar |
| uv | 最新 | Python 依赖管理（替代 pip） |
| Rust | 稳定版（含 `cargo`） | Tauri 2 桌面壳编译 |
| 操作系统 | macOS / Windows / Linux | Tauri 目标平台 |

> 所有数据存储于本地（SQLite + LanceDB），默认无需联网。

---

## 2. 获取代码

```bash
git clone <仓库地址> ibreeze
cd ibreeze
```

---

## 3. 安装依赖

### 3.1 前端（Tauri 桌面应用）

```bash
cd apps/desktop
npm install
```

### 3.2 后端（Python Sidecar）

```bash
cd sidecar
uv sync
```

`uv sync` 会依据 `pyproject.toml` 创建虚拟环境并安装全部依赖（含 pytest / pytest-asyncio 等开发依赖）。

---

## 4. 数据库迁移

后端使用 SQLite，迁移脚本位于 `sidecar/migrations/`（命名格式 `YYYYMMDDHHMMSS_description.sql`，当前范围 `0028`–`0061`）。迁移幂等（`CREATE TABLE IF NOT EXISTS` / 条件 `ADD COLUMN`）。

启动时 `acos.app` 自动执行未完成迁移。手动检查迁移状态：

```bash
# 通过 RPC（应用运行后）
rpcCall('sys.migration.status', {})
```

迁移失败时修复脚本后可直接重跑（幂等设计允许重跑）。

## 5. 本地开发

需同时启动后端 sidecar 与前端的 Tauri 外壳：

```bash
# 终端 1：后端 sidecar
cd sidecar
uv run python -m acos.app

# 终端 2：前端（含 Tauri dev，会拉起 Rust 外壳）
cd apps/desktop
npm run dev
```

RPC 链路：前端 `rpcCall` → Tauri 命令 `sys_rpc_call` → 通过 Unix Domain Socket 转 JSON-RPC 2.0（NDJSON）到 sidecar。

## 9. 常见问题

- **迁移报错**：确认 `sidecar/migrations/` 下脚本未被手动改动；修复后重跑 `uv run python -m acos.app` 即可。
- **端口 / socket 占用**：sidecar 使用 UDS，确保应用数据目录可写。
- **本地向量权重缺失**：知识库 embedding 采用本地确定性向量（环境无法下载 bge-m3 ONNX 权重时为兼容实现），不影响索引与 ACL 红线验证。
- **凭证存储**：Provider 凭证存于 OS Keychain，明文不落 SQLite。

## 11. 近期新增功能

- **Backend 健康探测**（迁移 0061）：`backends` 表新增 `last_health_probe_at`、`global_process_limit`、`plan_validator_config` 列；`BackendService.probe_health()` 自动探活 stale backends
- **PV-03 公司可配上限**：`companies.plan_validator_config` 支持 `max_nodes`、`max_depth`，按公司粒度控制任务 DAG 规模
- **Budget 降级分支**：`gov.budget.reserve` 支持 `on_exceeded="downgrade"` 策略，触顶后返回降级信号而非直接拒绝
- **task.nodes RPC**：返回 DAG 节点及依赖关系，前端 DAG 可视化使用
- **Task DAG 可视化**（React Flow）：`TaskDetail` 页面新增 DAG 标签页，展示节点依赖拓扑
- **RPC 返回值统一**：`session.list`、`session.transcript.get`、`provider.list`、`provider.model.list`、`org.grant.list` 均返回 `{items/grants/threads: [], total}` 包装格式；前端统一解包
- **RPC 错误处理**：`rpcCall` 新增 `result.error` 检测，后端 `org.grant.list` 的 `return {"error":...}` 改为抛异常
- **员工模板可绑定 Provider**：`TemplateList` 新增 Provider 下拉（从 `provider.list` 拉取），创建/编辑模板时写入 `provider_id` + `provider_type`；模板列表新增 Provider 列
- **员工可绑定基座模板**：`EmployeeList` 新增"基座模板"下拉（从 `org.template.list` 拉取），创建员工写入 `template_id`；员工列表新增"基座模型"列展示 `model · provider`
- **侧边栏 Dashboard 置顶**：默认首页改为 Dashboard；Header 补全所有页面标题（含 Dashboard / 会话 / Provider 等）
